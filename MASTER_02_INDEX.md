## 🔑 CONCEITOS-CHAVE

### 1. Multi-Tenant Isolation
**Leia em:** MASTER_02_ARCHITECTURE.md (Fluxo de Multi-Tenant)  
**Implementar em:** Fase 3

Cada request valida que o usuário tem acesso à empresa.  
Nenhuma query sem tenant filter.  
Impossível contornar isolamento.

### 2. RBAC Granular
**Leia em:** MASTER_02_ARCHITECTURE.md (Padrões de Segurança)  
**Implementar em:** Fase 4

10+ roles predefinidos (Admin, Manager, Viewer, etc).  
Permissões por resource + action (users:view, users:create, etc).  
Permission middleware em cada endpoint.

### 3. Auditoria Automática
**Leia em:** MASTER_02_ARCHITECTURE.md (Fluxos de Segurança)  
**Implementar em:** Fase 6

Logging automático de tudo (create, update, delete).  
Before/after values salvos.  
IP e device rastreados.  
Impossível apagar ou falsificar.

### 4. Feature Flags
**Leia em:** MASTER_02_IMPLEMENTATION_PLAN.md (Fase 8)  
**Implementar em:** Fase 8

Ativar/desativar features por empresa.  
Sem deploy necessário.  
A/B testing ready.  
Kill switch para emergências.

### 5. Billing Architecture
**Leia em:** MASTER_02_ARCHITECTURE.md (Modelo BillingPlan)  
**Implementar em:** Fase 8

Preparado para Stripe (sem cobrar ainda).  
Limites de recursos por plano.  
Upgrade/downgrade logic.  
Usage tracking.
